Variable Scope in Ruby

The scope of the variable dictates where in a program a variable is accessible. Ruby has four types of variable scope: local, global, instance, and class.


Local Variables


Local variables are limited to the block of code in which it is defined. For example, a local variable declared in a method cannot be accessed outside of the method. They are the simplest and most basic variable without any symbols attached to them. Local variable names begin with either an underscore or lower case letter.


Class Variables


These variables have @@ symbols preceding their names, and can be read and modified from any instance of the class in which the variable is defined. Only one variable value exists for all objects instantiated from this class. If one object instance changes the value of the variable, that new value will essentially change for all other object instances.


Instance Variables


Instance variables have a single @ sign before their name. These can be read and modified from the instance of the class in which it is created.


Global Variables


These have a $ symbol before the name and can be read and modified from anywhere in the Ruby doc, regardless of where they are declared. Use of global variables is strongly discouraged. The problem with global variables is that, not only are they visible anywhere in the code for a program, they can also be changed from anywhere in the application. This can make tracking bugs difficult.
