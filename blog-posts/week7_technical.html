What is SQL Injection?


SQL injection, while having been around for a long time, is still a very common vulnerability. In fact it was the #1 attack in 2010 and 2011. It occurs when commands are inserted into SQL where only data was expected. The injection is usually done on a login form.


When you sign up for a site, your information gets stored in various fields. When you login, the information you provide are queried against the site's database. If the information matches what is in the database, you are logged in.


However, what if the information you type in is something along the lines of "1=1," or you enter a comment demarcation to essentially disable a query parameter? The information in the field can include something like "DROP TABLE users," which would delete the table and screw everything to Hell.


So how do we go about preventing this? The answer is parameterized queries, and they look something like this:


"SELECT * FROM users WHERE email=? AND password=?", [user.email, user.password]


This tells the database that you will look up an email and its password, but you will pass the specific values later. The database then sets up a query plan, which is a set of steps to find the records.
