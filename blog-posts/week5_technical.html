Fun With Classes

In the world of Object Oriented programming, classes and objects model real-world objects intuitively.


All objects in a class are grouped together with similar object attributes. Instance methods within that class relate to the things that can be done by that specific class or group of objects.


Say I want to replicate professional skateboarders in Ruby. We would have a class known as Skateboarders. From there, I can make an instance of a class and call it Rodney Mullen. A particular skateboarder can be made from the Skateboarder class, which acts as a template, with attributes shared by skateboarders.


From there, we can have instance methods such as #ollie or #kickflip. Every instance of the Skateboarder class will be able to use these instance methods.


Classes contain information about attributes and methods. The first step is to define the initialize class, which creates the object. It accepts arguments that allow you to pass info about your object, such as height, nationality, and skateboard brand. These arguments can be saved as instance variables which can be accessed by the rest of the Class. Similarly, insance methods can be used by objects throughout the class.


Hope classes make a bit more sense now!
